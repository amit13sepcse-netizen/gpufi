Technical Analysis

üîç Query Plan Insights

Successful GPU Offload Patterns:

  • GpuScan: Custom Scan (GpuScan) on bench_numeric_xxlarge for filters and projections
  • GpuPreAgg: Parallel Custom Scan (GpuPreAgg) for GROUP BY operations
  • Scan Engine: VFS with GPU0; vfs=19230770, ntuples=1000000000
  • GPU Projection: All 18 columns projected on device when needed
  • GPU Scan Quals: Filter predicates evaluated on GPU (e.g., value2 BETWEEN 1000 AND 9000)
  • GPU Group Key: Category-based grouping executed on device

‚öôÔ∏è Configuration & Resolution

  • Initial Setup: Minimal pg_strom--6.0.sql without Arrow/Parquet FDW
  • Issue Discovered: Missing helper functions (nrows, pavg, psum) caused catalog corruption errors
  • Resolution Applied:
    • Restored full pg_strom--6.0.sql.orig
    • Created pgstrom.pavg(numeric) and pgstrom.psum(numeric) functions
    • Added boolean variants: pgstrom.pavg(numeric, bool), pgstrom.psum(numeric, bool)
    • Created aggregate support: pgstrom.favg_numeric, pgstrom.fsum_numeric
    • Implemented aggregates: pgstrom.avg_numeric(bytea), pgstrom.sum_numeric(bytea)
  • Cluster: PostgreSQL 15 on port 5434, systemd service with CUDA 12.3 environment
  • Result: All 4 test scenarios now fully operational with GPU acceleration
